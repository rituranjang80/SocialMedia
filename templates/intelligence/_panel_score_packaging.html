The thumbnail input was changed from a URL field to a multipart file upload because: - URL-based ingestion forced the server to fetch arbitrary URLs for content scoring, which is an SSRF surface unless every downstream consumer (Intelligence) sandboxes its fetches perfectly. Removing the URL field eliminates the class. - Server-side validation of an uploaded file is straightforward (size cap + Pillow magic-byte verification) and lets us guarantee Intelligence only ever sees real JPEG/PNG/WebP bytes with bounded size, neither true for an arbitrary URL. The form posts as multipart/form-data; the view base64-encodes the validated bytes server-side before forwarding to Intelligence. {% endcomment %}
